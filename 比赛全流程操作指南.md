## 一、开机准备

### 1.1 开机

正常启动电脑

### 1.2 确认硬件连接

- D435相机 → USB3.0
- Yesense IMU → USB
- STM32 → USB

***

## 二、启动系统（推荐：相机+IMU融合定位）

<br />

### 2.1 打开终端，设置ROS环境

```bash
cd ~/d435/catkin_ws
source devel/setup.bash
```

### 2.2 启动定位系统（推荐）

```bash
roslaunch d435_h30_localization d435_imu.launch send_to_stm32:=true
```

**说明**：

- `send_to_stm32:=true` - 同时启动串口通信，把位姿数据发给STM32
- `serial_assistant:=true` - 同时启动串口助手（默认开启），显示发送的数据和频率

**串口助手会显示**：

- 时间戳
- 数据序号
- 发送的数据内容
- 发送频率（Hz）
- 数据间隔（ms）

***

## 三、其他可选启动方式

### 3.1 只用相机（不用IMU）

```bash
roslaunch d435_h30_localization camera_only.launch send_to_stm32:=true
```

### 3.2 相机+IMU+CAD地图

```bash
roslaunch d435_h30_localization d435_imu_cad.launch send_to_stm32:=true
```

***

## 四、查看数据

### 4.1 直接在终端看（推荐！）

启动后终端会自动打印发送给STM32的数据：

```
[INFO] 发送数据: coor:123.5,67.8,45.2
```

格式说明：`coor:x（cm）,y（cm）,yaw（度）`

### 4.2 用RViz可视化

系统启动时会自动打开RViz，在里面可以看到：

- 相机图像
- 点云地图
- 定位轨迹

***

## 五、停止系统

在终端按：`Ctrl+C`

***

## 六、快速测试清单

```bash
# 1. 检查串口
ls /dev/ttyACM*
# 应该有 ttyACM0（IMU）和 ttyACM1（STM32）

# 2. 设置环境
cd ~/d435/catkin_ws
source devel/setup.bash

# 3. 启动系统
roslaunch d435_h30_localization d435_imu.launch send_to_stm32:=true

# 4. 在另一个终端确认节点正常
rosnode list
# 应该能看到 /rtabmap_pose_to_stm32
```

***

## 七、紧急：如果串口有问题

### 7.1 查看串口权限

```bash
ls -la /dev/ttyACM*
```

### 7.2 临时修复权限

```bash
sudo chmod 666 /dev/ttyACM*
```

***

## 八、数据格式说明

发送给STM32的数据格式：

```
coor:123.5,67.8,45.2
```

- `x` - x坐标（厘米）
- `y` - y坐标（厘米）
- `yaw` - 航向角（度）
- 波特率：115200

***

**记住**：必须加 `send_to_stm32:=true` 才会启动串口通信！
